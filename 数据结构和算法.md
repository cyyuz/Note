## 0.1 C语言的陷阱

- 内存重叠
- 临时变量太多或者没安全释放
- 没有测试内存越界
- 指针操作不熟悉

```c
void *memmove(void *dest, const void *src, size_t n)
{
    char *p1 = dest;
    conse char *p2 = src;
    if (p1 < p2) {  // dest在src前面
        while (n--) {
            *p1++ = *p2++;
        }
    } else {  // dest在src后面，内存可能重叠，因此从后往前拷贝
        p1 += n;
        p2 += n;
        while (n--) {
            *--p1 = *--p2;
        }
    }
    return dest;
}
```
